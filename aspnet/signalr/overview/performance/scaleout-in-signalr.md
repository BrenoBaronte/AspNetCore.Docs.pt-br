## <a name="implementation"></a><span data-ttu-id="6053e-140">Implementação</span><span class="sxs-lookup"><span data-stu-id="6053e-140">Implementation</span></span>

<span data-ttu-id="6053e-145">Cada instância de servidor se conecta ao backplane por meio do barramento.</span><span class="sxs-lookup"><span data-stu-id="6053e-145">Each server instance connects to the backplane through the bus.</span></span> <span data-ttu-id="6053e-146">Quando uma mensagem é enviada, ele passa para o backplane e backplane envia para todos os servidores.</span><span class="sxs-lookup"><span data-stu-id="6053e-146">When a message is sent, it goes to the backplane, and the backplane sends it to every server.</span></span> <span data-ttu-id="6053e-147">Quando um servidor recebe uma mensagem do backplane, ele coloca a mensagem em seu cache local.</span><span class="sxs-lookup"><span data-stu-id="6053e-147">When a server gets a message from the backplane, it puts the message in its local cache.</span></span> <span data-ttu-id="6053e-148">O servidor, em seguida, entrega mensagens para os clientes de seu cache local.</span><span class="sxs-lookup"><span data-stu-id="6053e-148">The server then delivers messages to clients from its local cache.</span></span>

<span data-ttu-id="6053e-149">Para cada conexão de cliente, progresso do cliente ao ler o fluxo de mensagem é controlado usando um cursor.</span><span class="sxs-lookup"><span data-stu-id="6053e-149">For each client connection, the client's progress in reading the message stream is tracked using a cursor.</span></span> <span data-ttu-id="6053e-150">(Um cursor representa uma posição no fluxo de mensagens.) - <span data-ttu-id="6053e-160">[Difusão de servidor](../getting-started/tutorial-server-broadcast-with-signalr.md) (por exemplo, cotações da bolsa): painéis posteriores funcionam bem para este cenário, porque o servidor controla a taxa na qual as mensagens são enviadas.</span><span class="sxs-lookup"><span data-stu-id="6053e-160">[Server broadcast](../getting-started/tutorial-server-broadcast-with-signalr.md) (e.g., stock ticker): Backplanes work well for this scenario, because the server controls the rate at which messages are sent.</span></span>
